Configuration Commands

This section describes various configuration commands supported by the BST feature. The name of the command becomes the method in the JSON-RPC message and any associated parameters for the command form the params in the JSON-RPC message.

All the commands are sent as HTTP 1.1 POST operations, with the JSON content as the HTTP message body.

Configure BST Thresholds

The configure-bst-thresholds command setups the BST thresholds for various realms in the Broadcom ASIC.

When a threshold is configured, and the associated buffer usage exceeds the threshold value, then an asynchronous notification is sent by the Agent.

The parameters associated with this commands depend on the realm for which the thresholds are being setup. The list of realms and associated parameters / thresholds for each realm is provided in API.

It may be noted that, at any point in time, the valid set of thresholds depend on the MMU configuration of the Broadcom ASIC. Reader is encouraged to consult the Broadcom ASIC documentation for supported combinations.

Threshold configuration is always in terms of bytes.

The associated REST URL for this command is as follows:

http://<ipaddr>:8080/broadview/bst/configure-bst-thresholds
